## Project Overview

TrendRadar is a lightweight news aggregation and filtering tool that monitors hot topics across multiple Chinese platforms (Weibo, Zhihu, Douyin, Baidu, etc.) and sends filtered notifications based on custom keywords.

**Key Components:**
- `main.py`: Core crawler and notification logic (5000+ lines)
- `mcp_server/`: FastMCP-based AI analysis server for querying news data
- `config/`: Configuration files (platforms, keywords, webhooks)
- `output/`: Generated news data (JSON) and HTML reports

**Tech Stack:** Python 3.10+, requests, PyYAML, pytz, FastMCP

## Common Commands

### Running the Crawler

```bash
# Install dependencies
pip install -r requirements.txt

# Run crawler once (fetches news, sends notifications, generates HTML)
python main.py

# The crawler will:
# 1. Fetch news from configured platforms (config/config.yaml)
# 2. Filter by keywords (config/frequency_words.txt)
# 3. Send notifications via configured channels
# 4. Save data to output/news_data.json and output/index.html
```

### MCP Server (AI Analysis)

```bash
# Install MCP server package
pip install -e .

# Run MCP server in stdio mode (for Claude Desktop, Cursor, etc.)
python -m mcp_server.server

# Run MCP server in HTTP mode (for remote access)
python -m mcp_server.server --http --port 3333

# Or use the convenience scripts:
# Windows: start-http.bat
# Mac/Linux: ./start-http.sh
```

### Docker Deployment

```bash
# Using docker-compose (from docker/ directory)
cd docker
docker-compose up -d

# View logs
docker logs -f trend-radar

# The container runs on a cron schedule (default: every 5 minutes)
# Set CRON_SCHEDULE env var to customize (e.g., "0 * * * *" for hourly)
```

### Testing GitHub Actions Locally

```bash
# Verify required config files exist
ls config/config.yaml config/frequency_words.txt

# Manually test the workflow steps
pip install -r requirements.txt
python main.py

# The workflow runs on schedule (default: hourly at :00)
# See .github/workflows/crawler.yml
```

## Architecture Overview

### Data Flow

1. **News Fetching** (`main.py:fetch_hot_news()`)
   - Fetches from newsnow API (https://api.newsnow.busiyi.world)
   - Supports 40+ platforms configured in `config/config.yaml`
   - Each platform returns top ~50 news items with title, URL, rank

2. **Keyword Matching** (`main.py:filter_news_by_frequency_words()`)
   - Reads keywords from `config/frequency_words.txt`
   - Supports 3 syntax types: regular words, must-have (+), exclusion (!)
   - Groups keywords by blank lines for independent tracking
   - Example: `华为\nOPPO\n+手机\n!广告` matches news with (华为 OR OPPO) AND 手机 AND NOT 广告

3. **Ranking Algorithm** (`main.py:calculate_news_score()`)
   - Weighted scoring: rank (60%) + frequency (30%) + hotness (10%)
   - Configurable weights in `config/config.yaml`
   - Tracks news history across time for trend analysis

4. **Notification Dispatch** (`main.py:send_notification()`)
   - Supports 6 channels: Feishu, DingTalk, WeCom, Telegram, Email, ntfy
   - Three push modes:
     - `daily`: All matched news from today
     - `current`: Latest batch matched news
     - `incremental`: Only new news since last run
   - Webhooks configured in `config/config.yaml` or GitHub Secrets (for Actions)

5. **Data Persistence**
   - `output/news_data.json`: Complete news history with timestamps
   - `output/index.html`: Pretty HTML report (GitHub Pages compatible)
   - `output/push_record.json`: Tracks sent notifications to avoid duplicates

### MCP Server Architecture

The `mcp_server/` directory provides AI-powered news analysis via Model Context Protocol:

**Structure:**
- `server.py`: FastMCP app with 13+ tool endpoints
- `services/`: Data access layer
  - `data_service.py`: Fetches and filters news from JSON files
  - `parser_service.py`: Parses news_data.json structure
  - `cache_service.py`: In-memory caching for performance
- `tools/`: MCP tool implementations (analytics, search, system management)
- `utils/`: Date parsing, validation, error handling

**Key Tools:**
- `get_latest_news()`: Fetch recent news by platform/date
- `search_news()`: Multi-field search (title, platform, date range)
- `analyze_topic_trend()`: Track keyword popularity over time
- `get_platform_statistics()`: Compare activity across platforms
- `generate_smart_summary()`: AI-friendly news summaries

**Data Source:** All tools read from `output/news_data.json` which is generated by `main.py`

### Configuration System

**config/config.yaml** - Main configuration
- `crawler.enable_crawler`: Master switch (set false to disable)
- `report.mode`: Push strategy (daily/current/incremental)
- `notification.webhooks`: Channel credentials (keep in GitHub Secrets for public repos)
- `platforms`: List of news sources to monitor
- `weight`: Ranking algorithm parameters

**config/frequency_words.txt** - Keyword filters
- One keyword per line
- Prefix `+` for required terms
- Prefix `!` for exclusions
- Blank lines separate keyword groups
- Groups are ranked independently in notifications

### Environment Variables (Docker & GitHub Actions)

Core config can be overridden via env vars:
- `ENABLE_CRAWLER`: true/false
- `ENABLE_NOTIFICATION`: true/false
- `REPORT_MODE`: daily/current/incremental
- `FEISHU_WEBHOOK_URL`, `TELEGRAM_BOT_TOKEN`, etc.: Channel credentials
- `CRON_SCHEDULE`: Docker cron expression (default: `*/5 * * * *`)

See `docker/.env` for full list.

## Development Guidelines

### Modifying Crawler Logic

The `main.py` file is monolithic (5000+ lines) but well-structured:

1. **Adding new platforms**: Update `config/config.yaml` platforms list (IDs from newsnow API)
2. **Changing notification format**: Modify `format_*_message()` functions (line ~1500+)
3. **Adjusting ranking**: Edit `calculate_news_score()` and `weight` in config.yaml
4. **New notification channels**: Add handler in `send_notification()` (line ~800+)

### Extending MCP Server

1. **New tools**: Add `@mcp.tool` decorated function in `mcp_server/server.py`
2. **New data sources**: Extend `DataService` in `services/data_service.py`
3. **New analysis**: Add methods in `tools/analytics.py`

All MCP tools should:
- Return JSON strings (use `json.dumps(result, ensure_ascii=False)`)
- Accept optional `project_root` for testing
- Include detailed docstrings (shown to AI clients)
- Handle errors gracefully (use custom exceptions from `utils/errors.py`)

### Testing Changes

**No formal test suite exists.** Manual testing approach:

1. **Crawler**: Run `python main.py` and check `output/` files
2. **Config changes**: Verify with small keyword sets first
3. **MCP tools**: Test via Claude Desktop or `python -c "from mcp_server.tools import ..."`
4. **Notifications**: Use test webhooks (create separate bots for testing)

### Deployment Considerations

**GitHub Actions:**
- Runs on public GitHub resources (limited quota per account)
- Commits generated files back to repo
- Use GitHub Secrets for webhooks (never commit to config.yaml)
- Cron scheduling has 5-10 min variance

**Docker:**
- Preferred for precise scheduling and high-frequency updates
- Volumes: Mount `config/` (read-only) and `output/` (read-write)
- Use `.env` file for config overrides
- Runs as cron daemon by default (set `RUN_MODE=once` for single execution)

**GitHub Pages:**
- Automatically deploys `output/index.html` if Pages enabled
- Enable in Settings → Pages → Source: master branch / root
- Updates on every crawler run via Actions commit

## Important Notes

- **Webhook Security**: Never commit webhooks to public repos. Use GitHub Secrets or `.env` files
- **API Rate Limits**: The newsnow API has no documented limits, but avoid intervals < 30 min on GitHub Actions
- **Data Format**: `output/news_data.json` structure is critical for MCP server. Don't modify schema without updating parsers
- **Chinese Content**: All news data, keywords, and docs are in Chinese. Keep this context when adding features
- **Time Zone**: All times use Asia/Shanghai (UTC+8) via pytz

## Quick Start for New Contributors

1. Fork the repository
2. Copy `config/config.yaml` and `config/frequency_words.txt` (add your platforms/keywords)
3. Set GitHub Secrets for notification webhooks (optional)
4. Enable GitHub Actions in your fork
5. Wait for the hourly cron run or trigger manually via Actions → Run workflow
6. Check `output/index.html` via GitHub Pages or locally

For MCP server development:
1. Install: `pip install -e .`
2. Generate test data: `python main.py` (creates `output/news_data.json`)
3. Start server: `python -m mcp_server.server`
4. Test with Claude Desktop or Cherry Studio (see `README-Cherry-Studio.md`)
